（2）需要用链表来表示打开文件的窗口（c语言的习惯用法），不然的花卉遇到诸多困难，例如窗口的增、删、改，
成员所占内存的即时分配和删除等，链表符合此需求，单也是碰了壁才改代码的，遇到不少麻烦。


（3）测试是发现结构体链表，如果结构体成员里有QString 变量，在调用append方法时出错，猜测原因是：QString 是一个
类名，在使用 malloc函数动态分配内存时，只分配了内存，没有分配相应的方法。网上查找相关问题，有提到使用 new 关键词。
面对此问题，我尝试用C++类来代替结构体。

（4）类释放内存用的时析构函数，普通变量自动消亡，但在例子中，
MainWindow::~MainWindow()
{
    delete ui;
}
需要指定删除ui控件，猜测控件需要手动删除。
用new / delete  来取代  malloc / free










==============================备忘
（1）按钮点击事件传参问题
普通的槽函数如：connect(pushbutton, SIGNAL(clicked()), this , SLOT(btnOnClick()));
无法传参，在这点我纠结了很久，也没找到方法，于是在网上查找相关内容。
找到某位网友提出用  lambda表达式，
【

Lambda 表达式，也可称为闭包，它是推动 Java 8 发布的最重要新特性。

Lambda 允许把函数作为一个方法的参数（函数作为参数传递进方法中）。

使用 Lambda 表达式可以使代码变的更加简洁紧凑。

】

QT支持lambda

例子：
connect(ui->pushButton_3, &QPushButton::clicked, this, [=](){btn3_onClick(2);});
其中 2 是传递的参数值

====================================================终于到了完结的时候了
虽然大工基本告成，心里挺高兴的。历时近一个星期，花了我大部分的时间、精力，虽然困难重重，但是不负最初的想法，
将基本的功能一一实现。虽然还有“用户登录”、“网络服务器同步”等功能没有实现，想必不会比之前的过程简单，只能留
待后面去拓展了，现在有点劳累。
在编程的过程中，还是遇到了不少的问题，也有所感想，零零碎碎，现在逐一记录：
（1）以简要为主
在构思整个方案的时候，常常有不切实际的计划，比如“浏览文件的界面”设计，要实现好的用户体验，但是真正编程的时候困
难重重。所以，为了突出实际的效用，应该把关注点放在基本的功能实现上，尽量简洁，而其他的完善想法可以留待后面去增添。
（2）关于面向对象思想，有了点体会
在刚开始编程的时候，对类的成员的作用理解还是不深刻的，所以写出的代码有很多自己都不愿重新检查一遍，逻辑、框架上不
够明了。特别是我在设计文件编辑tabWidget界面时，由于一些逻辑上的困难，最终选择了“类链表”的方法来描述一组“对象”，
是c语言结构体链表的翻版，但估计我也是应用了c++ 面向对象思想 的一些皮毛。
（3）层次结构
代码层次结构有的地方比较混乱，例如 “文件关闭” 功能，有几个一级级调用的函数过程，和“文件打开”等并列的功能方式没有很统一。
（4）重写想法
由于意识到构建工程中遇到的诸多问题，所以想要重建一个工程，以作优化。不过工程量有点大，所以先放着，留待以后再来着手一试。
现在的话。可以先出一个版本，在gitlab上搞一个分支。
（5）QT编程中，遇到的很多问题通过网上查找知识，得到了解决，也总结了不少方法，放置于备忘录上，以供查阅。
（6）关于该应用的日记功能
有不少的瑕疵支出==之处，和我预想的有些差距，但是勉强可以一用。我的最初想法是：开发一个真正可供自己写日记的应用，我也会去使
用它，但是现在的情况下，我会考虑姑且试用一下，有空自己再参考此工程，再新建一个。再看了看有道云笔记的界面和功能，我发现自己
